### **Packet Tracer - Configuring Extended ACLs (Step-by-Step Guide)**  
This guide will help you **configure, apply, and verify** **Extended ACLs** in **Cisco Packet Tracer**.

---

## **üìå Scenario**
- **PC1** should only have **FTP** and **ICMP (ping)** access to the **Server**.
- **PC2** should only have **HTTP** and **ICMP (ping)** access to the **Server**.
- **PC1 and PC2 should not be able to communicate with each other**.
- We will configure **two Extended ACLs**:
  - **Numbered ACL (100) for PC1**
  - **Named ACL (HTTP_ONLY) for PC2**.

---

1Ô∏è‚É£ **Open Cisco Packet Tracer**.  
2Ô∏è‚É£ **Add the following devices**:
   - 1 **Router (R1)**

3Ô∏è‚É£ **Connect them with cables**:
   - **G0/0 (R1) ‚Üí PC1**
   - **G0/1 (R1) ‚Üí PC2**
   - **G0/2 (R1) ‚Üí Server**
   
---

## **Step 2: Configure IP Addresses on R1**
1Ô∏è‚É£ **Open R1 CLI** and enter **configuration mode**:

   ```bash
   enable
   configure terminal
   ```

2Ô∏è‚É£ **Assign IP addresses to interfaces**:

   ```bash

   ip address 172.22.34.65 255.255.255.224

   exit

   ip address 172.22.34.97 255.255.255.240
   no shutdown
   exit

   interface gigabitEthernet 0/2
   ip address 172.22.34.1 255.255.255.192
   no shutdown
   exit
   ```

2Ô∏è‚É£ **PC2 Configuration**:

3Ô∏è‚É£ **Server Configuration**:

---

## **Step 4: Configure Numbered Extended ACL for PC1 (FTP + ICMP)**
1Ô∏è‚É£ Open **R1 CLI** and enter:

   ```bash
   configure terminal
   ```

2Ô∏è‚É£ Create **Extended ACL 100** to allow **FTP and ICMP**:

   ```bash
   access-list 100 permit tcp 172.22.34.64 0.0.0.31 host 172.22.34.62 eq ftp
   access-list 100 permit icmp 172.22.34.64 0.0.0.31 host 172.22.34.62
   exit
   ```

3Ô∏è‚É£ **Apply ACL 100** to **G0/0** (Inbound Traffic):

   ```bash
   interface gigabitEthernet 0/0
   ip access-group 100 in
   exit
   ```

---

## **Step 5: Configure Named Extended ACL for PC2 (HTTP + ICMP)**
1Ô∏è‚É£ Open **R1 CLI** and enter:

   ```bash
   configure terminal
   ```

2Ô∏è‚É£ Create **Extended Named ACL**:

   ```bash
   ip access-list extended HTTP_ONLY
   permit tcp 172.22.34.96 0.0.0.15 host 172.22.34.62 eq www
   permit icmp 172.22.34.96 0.0.0.15 host 172.22.34.62
   exit
   ```

3Ô∏è‚É£ **Apply ACL to Interface G0/1**:

   ```bash
   interface gigabitEthernet 0/1
   ip access-group HTTP_ONLY in
   exit
   ```
